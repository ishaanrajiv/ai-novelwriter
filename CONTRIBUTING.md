# Contributing

Thanks for your interest in contributing.

## Getting Started

1. Fork the repository and create a feature branch.
2. Install dependencies with `bun install`.
3. Run checks locally:
   - `bun run typecheck`
   - `bun test`
4. Open a pull request with a clear description of the change and why it is needed.

## Development Notes

- Keep changes focused and small when possible.
- Add or update tests for behavior changes.
- Update documentation for any user-facing CLI or config changes.

## Pull Request Checklist

1. Tests pass locally.
2. Type checks pass locally.
3. Docs and examples are updated when relevant.
4. No secrets or local environment files are included.
